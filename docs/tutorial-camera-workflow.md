# Tutorial: Camera workflow parity

This tutorial walks through the complete Alice camera workflow from the local
REST API. You will launch a project, move the camera, apply a view preset, save
a marker, restore it, delete it, and switch to first-person mode.

## What you need

- Node.js
- npm
- curl
- A checkout of the Alice web repository

## 1. Build and start Alice

```bash
npm install
NODE_OPTIONS=--max-old-space-size=32768 npm run build:server
export ALICE_LOCAL_API_TOKEN="$(node -e 'console.log(require("crypto").randomBytes(32).toString("base64url"))')"
npm run serve -- --port 3099 --evidence-dir ./evidence --api-token "$ALICE_LOCAL_API_TOKEN"
```

Keep the server running. Use a second terminal for the remaining commands and
export the same token there:

```bash
export ALICE_LOCAL_API_TOKEN="<same value used to start alice-web>"
export ALICE_WEB_URL=http://127.0.0.1:3099
```

## 2. Launch a scene

```bash
curl -X POST "$ALICE_WEB_URL/api/launch" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{}'
```

Expected response shape:

```json
{
  "status": "launched",
  "projectName": "Program",
  "sceneObjectCount": 2
}
```

The default launch creates an Alice scene with a ground and camera.

## 3. Read the starting camera state

```bash
curl "$ALICE_WEB_URL/api/camera/state" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN"
```

Expected response shape:

```json
{
  "schema_version": "eatme.alice-camera-workflow-state/v1",
  "status": "ok",
  "operation": "state",
  "camera": {
    "mode": "orbit",
    "position": { "x": 0, "y": 5, "z": 20 },
    "target": { "x": 0, "y": 1, "z": 0 },
    "fieldOfViewDegrees": 60,
    "activePreset": "home"
  },
  "markers": [],
  "activeMarkerId": null
}
```

`yawDegrees`, `pitchDegrees`, `rollDegrees`, and `up` are also included in the
full response.

## 4. Move the camera

Move forward two Alice scene units:

```bash
curl -X POST "$ALICE_WEB_URL/api/camera/move" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"forward":2}'
```

The response returns the updated camera state with `operation` set to `move`.
In orbit mode, the camera position and target move together. The response also
sets `activePreset` to `null` because the moved camera no longer exactly
matches the `home` preset.

## 5. Apply a view preset

Switch to the front view:

```bash
curl -X POST "$ALICE_WEB_URL/api/camera/preset" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"preset":"front"}'
```

Check that the response camera includes:

```json
{
  "activePreset": "front"
}
```

## 6. Save a marker

Save the front view as a marker and keep the response:

```bash
curl -X POST "$ALICE_WEB_URL/api/camera/markers" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"name":"Intro front view"}' \
  > /tmp/alice-camera-marker.json
```

Expected response shape:

```json
{
  "schema_version": "eatme.alice-camera-workflow-state/v1",
  "status": "ok",
  "operation": "save-marker",
  "marker": {
    "id": "camera-marker-1",
    "name": "Intro front view",
    "camera": {
      "mode": "orbit",
      "activePreset": "front"
    }
  },
  "markers": [
    {
      "id": "camera-marker-1",
      "name": "Intro front view",
      "camera": {
        "mode": "orbit",
        "activePreset": "front"
      }
    }
  ],
  "activeMarkerId": "camera-marker-1"
}
```

Save the marker ID from the response:

```bash
export CAMERA_MARKER_ID="$(
  node -e 'const fs=require("fs"); const data=JSON.parse(fs.readFileSync("/tmp/alice-camera-marker.json", "utf8")); console.log(data.marker.id);'
)"
```

The marker ID is generated by Alice. Treat it as an opaque string.

## 7. Move away from the marker

Switch to an isometric view:

```bash
curl -X POST "$ALICE_WEB_URL/api/camera/preset" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"preset":"isometric"}'
```

The saved marker remains in the marker list.

## 8. Restore the marker

```bash
curl -X POST "$ALICE_WEB_URL/api/camera/markers/$CAMERA_MARKER_ID/restore" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{}'
```

The response camera returns to the saved front view and `activeMarkerId` matches
the marker ID.

## 9. Delete the marker

```bash
curl -X DELETE "$ALICE_WEB_URL/api/camera/markers/$CAMERA_MARKER_ID" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN"
```

The deleted marker no longer appears in `markers`.

## 10. Switch to first-person mode

```bash
curl -X POST "$ALICE_WEB_URL/api/camera/mode" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"mode":"first-person"}'
```

Expected response shape:

```json
{
  "operation": "mode",
  "camera": {
    "mode": "first-person"
  }
}
```

Move forward in first-person mode:

```bash
curl -X POST "$ALICE_WEB_URL/api/camera/move" \
  -H "X-Alice-Local-Api-Token: $ALICE_LOCAL_API_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"forward":1}'
```

In first-person mode, movement is from the camera's point of view and the target
is recalculated from yaw and pitch.

## 11. Try the same flow in the browser

Start the browser app:

```bash
NODE_OPTIONS=--max-old-space-size=32768 npm run build
npm run dev
```

Use the **Camera** panel to repeat the same flow:

1. Move forward.
2. Select the front view.
3. Save a marker named `Intro front view`.
4. Select the isometric view.
5. Restore the marker.
6. Delete the marker.
7. Toggle first-person mode.

The browser status region reports each action in plain Alice wording.
